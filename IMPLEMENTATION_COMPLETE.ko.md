### 구현된 취약점

1. **Envoy 프록시 (envoy.yaml)**:
   - ✅ `allow_connect: true` - HTTP/2 CONNECT 활성화
   - ✅ `connect_matcher: {}` - IP 필터링 없음 (모든 대상 수락)
   - ✅ 내부 IP를 차단하는 RBAC 정책 없음
   - ✅ 임의의 대상으로의 동적 포워드 프록시

2. **백엔드 서비스 (app.py)**:
   - ✅ `:authority`에서 파생된 `Host` 헤더 신뢰
   - ✅ 부분 문자열 일치: `host.lower()`에 `'internal.acme.corp'` 포함 여부 확인
   - ✅ 소스 IP 검증 없음
   - ✅ 인증 불필요
   - ✅ `/admin` 엔드포인트에 플래그 노출

### 공격 흐름

```
1. 참가자가 HTTP/2 지원 확인
2. CONNECT 메서드 테스트 → 작동 확인
3. 내부 네트워크 (172.20.0.0/24) 스캔
4. 172.20.0.10:8080에서 백엔드 발견
5. 엔드포인트 열거 → /admin이 403 반환 확인
6. :authority 헤더 값 테스트
7. :authority = internal.acme.corp 사용
8. 403 우회 → 플래그 획득
```

## 문제 해결 (Troubleshooting)

### 흔한 문제

**문제**: 10000 포트가 이미 사용 중임
**해결책**: SETUP.md의 "문제 해결" 섹션 참조 → docker-compose.yml에서 포트 변경

**문제**: 컨테이너가 시작되지 않음
**해결책**: `docker-compose logs`로 로그 확인, Docker 데몬 실행 여부 확인

**문제**: 프록시에서 백엔드로 연결할 수 없음
**해결책**: `docker network inspect`로 네트워크 확인, 백엔드 IP 확인

**문제**: 플래그가 나타나지 않음
**해결책**: `docker exec backend.acme.corp env | grep FLAG`로 FLAG 환경 변수 확인

**전체 문제 해결 가이드**: `SETUP.md` 참조
